Implementation Plan Agent

This step requires knowledge of your codebase, so you must use an AI coding assistant such as Cursor, Claude Code, or GitHub Copilot.

The Implementation Plan Agent checks for problems before coding begins. It compares the Journey Specification with your codebase and identifies errors and conflicts between new designs and existing code. Resolving these issues early prevents problems during implementation.

Pass your Journey Specification from the previous step to your AI coding assistant using the Implementation Plan Agent prompt.

Review

Review the summary output from the agent. It lists the most important changes and action points. If you are happy, review the new implementation plan file. Edit the implementation plan manually if you disagree with the build order.

Mistakes are easier to spot and fix at this stage than during the coding stage. If you see an issue you can fix, suggest an update to the agent so it helps others in future.

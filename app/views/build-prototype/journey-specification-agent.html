Journey Specification Agent

The Journey Specification Agent takes design artefacts and converts them into screen templates.

Designs often have gaps, or assume the developer already understands certain decisions. The Coding Agent should only write code, not fill gaps or make product decisions. So the Journey Specification Agent fills those gaps first and returns the journey in a format that is easy to review. You review the output before passing to the next agent, so the solution is complete before implementation begins.

Customise this for your project. Tell the agent where your designs come from.

Copy and paste plain text descriptions of the screens and user journey.

Other designs

You can use any format for your designs, including images, screenshots and user stories. Include all the files the agent needs. If something does not work, ask in #ask-ace for support.

Analyse

Use a powerful AI agent to create the specification file. Complex tasks work best with more powerful models. Copy the Journey Specification Agent prompt and add your designs.

Review

Review the output carefully. Start with the Decision Log to see the most critical decisions. Then go screen by screen, checking the order is correct. Finally, review the details for each screen.

Friction is data. If you spot mistakes at this stage, the final result will be better. If you can update the agent prompt, the fix benefits everyone who uses this workflow after you.

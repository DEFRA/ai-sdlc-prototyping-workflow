How the workflow works

Each AI agent performs a specific task. You review the output from the agent, then pass the task on to the next agent. When something goes wrong, fix the agent so it works better next time.

The steps are:

  1. Start with designs
  2. Run the Journey Specification Agent
  3. Run the Implementation Plan Agent
  4. Run the Coding Agent
  5. Deploy

What each agent does

Journey Specification Agent turns designs into screen templates. It fills in gaps from the designs and highlights key implementation decisions for you to review.

Implementation Plan Agent compares the journey specification and designs to the existing codebase. It helps to identify conflicts in the new designs, and checks that the new screens follow existing patterns.

Coding Agent writes code following your repository's rules and patterns, using the implementation plan to guide each change.
